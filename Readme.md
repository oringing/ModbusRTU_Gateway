# Modbus RTU 网关（STM32F103C8T6）

> **版本**: v1.9.0 | **许可证**: MIT | **最后更新**: 2026-05-17  
> 基于 STM32F1 + FreeRTOS 的工业级 Modbus RTU 协议从机网关，具备高稳定性 UART 接收链、故障自恢复机制、双路异构舵机控制与系统心跳监控。

## 🔍 技术亮点

### ✅  UART 接收链防护

- **背靠背请求帧丢失防护**：通过空闲中断（IDLE）+ 双缓冲 + HAL 状态机同步重绑，消除奇偶交替响应故障
- **错误分级恢复机制**：ORE 立即恢复、FE/NE 连续 3 次触发恢复、PE 连续 5 次触发恢复，避免误判导致频繁重启
- **硬件故障熔断保护**：恢复重试超限（10 次）后标记硬件故障，进入系统降级运行，连续 5 帧正常后自动恢复

### ✅ 故障预警与自恢复

- **独立看门狗（IWDG）**：2 秒超时复位（LSI 40kHz / 预分频 256 / 重载值 312），修正 LSI 时钟开启与重载值计算陷阱
- **系统心跳监控**：寄存器 `0x0003` 初始值 `0x3000`，Bit 0~3 每秒翻转（`0x3000 ↔ 0x300F`），Bit 4~7 固定保留系统状态
- **栈水位监控**：周期性检测任务栈高水位（60s 间隔），低水位告警与恢复通知机制，预防栈溢出风险
- **安全模式停机**：配置校验 fail-fast，LED 编码指示 HardFault/MemManage/BusFault 等严重故障

### ✅ 协议层防护

- **协议层入参校验**：在 `Modbus_Process()` 入口处校验帧长度、从机地址、CRC，非法帧直接丢弃
- **寄存器访问边界保护**：读/写请求中校验起始地址与数量组合，防止数组越界和整数溢出
- **寄存器互斥保护**：单次加锁快照读取，写回调锁外执行，避免并发一致性风险
- **日志限流机制**：错误日志按频率控制输出，避免阻塞主流程

### ✅ 双路异构舵机控制

- **180° 舵机（SG90）**：地址 `0x0004`，通过 Modbus 写寄存器实现 0°~180° 精确角度控制（TIM3_CH1/PA6，50Hz PWM）
- **360° 连续旋转舵机**：地址 `0x0005`，通过 Modbus 写寄存器实现速度/方向控制（TIM3_CH2/PA7）
  - `0~126`：反转调速（值越小反转越快）
  - `127`：中位停止（死区）
  - `128~255`：正转调速（值越大正转越快）
- **脉宽级精确控制**：BSP 层提供微秒级脉宽控制接口，消除占空比切换毛刺影响

### ✅ 代码工程化规范

- **魔法数字清理**：消除全项目硬编码索引与业务阈值，22 个 Modbus 帧索引宏分组管理，18 个栈/缓冲区校验阈值提取为宏
- **注释规范化**：全项目 25 个文件完成 L1-L4 分级注释（公共 API Doxygen 注释 + static 函数设计意图注释 + 物理参数单位说明）
- **代码格式化**：基于 Linux 内核风格 + MISRA 约束的 .clang-format 配置，4 空格缩进、K&R 大括号风格、100 字符列宽限制

## ️ 技术栈

- **MCU**: STM32F103C8T6 (ARM Cortex-M3, 72MHz, 64KB Flash, 20KB RAM)
- **RTOS**: FreeRTOS v10.3.1 (CMSIS-RTOS v1 接口)
- **通信协议**: Modbus RTU over RS485 (MAX485, 自动流向控制型)
- **固件库**: STM32 HAL (F1)
- **工具链**: STM32CubeMX + Keil MDK v5 + VS Code
- **代码规范**:注释分级规范 v2.2 + .clang-format（Linux 内核风格）

### 2. TTL 转 RS485 模块（自动流向控制型）接线说明

| 模块引脚   | 连接 STM32或USB-TTL  | 说明              |
| ------ | ----------------- | ----------------- |
| VCC    | 3.3V              | 模块电源（3.3V/5V兼容） |
| GND    | GND               | 共地                   |
| TXD    | PA9 (USART1\_TX)  | 模块发送接MCU发送       |
| RXD    | PA10 (USART1\_RX) | 模块接收接MCU接收       |
| A      | A+ (主机侧)        | 485总线正极            |
| B      | B- (主机侧)        | 485总线负极            |
| 大地/屏蔽地 | （悬空）       | 接大地端，抗干扰        |

```
STM32F103从机     MAX485模块（从机端）    MAX485模块（主机端）         PC USB转TTL    
┌─────────┐        ┌──────────┐           ┌──────────┐             ┌──────────┐ 
│ PA9  TX ├───────►│ TXD   A+ │◄─────────►│ A+   TXD │◄────────────│ TXD      │
│ PA10 RX │◄───────│ RXD      │           │      RXD │────────────►│ RXD      │
│ 3.3V    ├────────│ VCC   B- │◄─────────►│ B-   VCC │─────────────│ 3.3V     │
│ GND     ├────────│ GND──────│───────────│──────GND │─────────────│ GND      │
└─────────┘        └─────────┬┘           └┬─────────┘             └──────────┘
                           屏蔽地（悬空） 屏蔽地（悬空）

```

- 务必确保两个max485模块的VCC都接入3V3，禁止直接连接5V，否则有损坏stm32的风险。
- 主机侧和从机侧MAX485模块的GND建议共地，避免信号干扰。
- 本模块为自动流向控制型，无需MCU控制RE/DE引脚，操作与普通串口一致。
- MCU的UART串口 或 PC的USB转TTL串口 的 TX 连接 本模块串口侧 的 TXD 引脚，RX同理，否则无法正常通信。

### 3. 舵机 PWM 接口（已接入）

| 引脚 | 功能 | 说明 |
|------|------|------|
| PA6 | TIM3_CH1 (180° 舵机) | 50Hz PWM，脉宽 0.5ms~2.5ms |
| PA7 | TIM3_CH2 (360° 舵机) | 50Hz PWM，脉宽 0.5ms~2.5ms |
| 5V（独立电源） | 舵机供电 | **必须独立供电，严禁与 MCU 共用 3.3V** |
| GND | 共地 | 5V 电源 GND，stm32 GND三者必须共地 |


**关键接线说明**：

| 连接类型 | 路径 | 注意事项 |
|---------|------|---------|
| **PWM 信号** | PA6/PA7 → 舵机信号线 | 杜邦线直连，长度 ≤ 15cm |
| **舵机供电** | 外部 5V 电源 → 舵机 VCC | **严禁接 MCU 3.3V** |
| **共地要求** | 外部 5V GND → STM32 GND | **必须共地，否则 PWM 无效** |

**舵机接线示意图**：
```
STM32F103              舵机              外部电源
┌───────┐       ┌────────────────┐     ┌───────┐  
│   PA6 │◄─────►│ 180° 舵机信号线 │     │       │
│   PA7 │◄─────►│ 360° 舵机信号线 │     │       │
│       │       │  5V （独立供电）│─────│ 5V  + │
│   GND ├───────│ GND ───────────│─────│ GND - │
└───────┘       └────────────────┘     └───────┘    
```

**供电警告**：
- ⚠️ **舵机 VCC 必须接外部 5V 电源**（如 LM2596 降压模块、5V 电池或独立电源适配器）
- ❌ **严禁接 MCU 的 3.3V 引脚**（舵机启动电流峰值 500mA~800mA，MCU 仅能提供 200mA）
- ✅ **GND 必须共地**（5V 电源 GND 与 STM32 GND 连接，否则 PWM 电平参考点不一致）

**PWM 特性**：
- 周期：20ms（50Hz）
- 高电平脉宽：0.5ms（0°）~ 2.5ms（180°）
- 中位死区：1.5ms 对应 360° 舵机停止状态

### 4. I2C 传感器接口（待接入）

| 引脚 | 功能 | 说明 |
|------|------|------|
| PB6 | I2C1_SCL | AHT20/BMP280 时钟线 |
| PB7 | I2C1_SDA | AHT20/BMP280 数据线 |
| 3.3V | VCC | 传感器供电（注意电平匹配） |
| GND | GND | 共地 |


**AHT20/BMP280 接线示意图**：
```
STM32F103      AHT20/BMP280 传感器模块
┌─────────┐       ┌────────┐
│ PB6  SCL├──────►│ SCL    │
│ PB7  SDA│◄─────►│ SDA    │
│ 3.3V    ├──────►│ VCC    │
│ GND     ├───────│ GND    │
└─────────┘       └────────┘
```

**I2C 总线特性**：
- 速率：100kHz（标准模式）
- 上拉电阻：硬件 I2C 控制器内置，无需外部上拉
- 设备地址：AHT20 (0x38)，BMP280 (0x76/0x77，取决于 SDO 引脚)
- 挂载数量：同一总线最多支持 2 个传感器（地址不冲突）

**上电时序注意事项**：
- AHT20：上电后需等待 20ms 完成初始化，首次读取前发送 `0x71` 状态查询命令
- BMP280：上电后需等待 10ms，首次读取前配置 `oversampling` 和 `mode` 寄存器
- 总线挂载多个设备时，确保所有传感器 VCC/GND 连接正确后再上电


**调试建议**：
- 使用 I2C 扫描工具确认设备地址（STM32CubeProgrammer 或 Arduino Wire 库）
- 首次通信前检查 SCL/SDA 引脚电平（空闲时应为高电平 3.3V）
- 若读取返回 `0xFF` 或超时，检查接线极性、上拉电阻、设备地址是否正确

### 5. 关键接线要点（务必检查）

- **USART1**：PA9 (TX)、PA10 (RX)

## ️ 寄存器映射

### 保持寄存器地址表（10 个）

| 地址 | 宏定义 | 功能说明 | 读写 | 默认值 | 联动功能 |
|------|--------|----------|------|--------|----------|
| 0x0000 | `MODBUS_REG_ADDR_SENSOR_TEMP` | AHT20 温度（预留） | R | 0x0000 | 待接入 AHT20 |
| 0x0001 | `MODBUS_REG_ADDR_SENSOR_HUMIDITY` | AHT20 湿度（预留） | R | 0x1000 | 待接入 AHT20 |
| 0x0002 | `MODBUS_REG_ADDR_SENSOR_PRESSURE` | BMP280 气压（预留） | R | 0x2000 | 待接入 BMP280 |
| 0x0003 | `MODBUS_REG_ADDR_DEVICE_STATUS` | **系统状态寄存器** | R | 0x3000 | 心跳+故障标志 |
| 0x0004 | `MODBUS_REG_ADDR_SERVO_TARGET` | **180° 舵机目标角度** | **W** | 0x4000 | TIM3_CH1/PA6 |
| 0x0005 | `MODBUS_REG_ADDR_SERVO_SPEED` | **360° 舵机速度/方向** | **W** | 0x0000 | TIM3_CH2/PA7 |
| 0x0006~0x0009 | - | 预留扩展 | R | 0x0000 | 未来功能扩展 |

**0x0003 系统状态寄存器位定义**：
- **默认初始值**: `0x3000`（上电复位后的初始状态）
- **Bit 0~3**: 心跳计数器（每秒翻转，初始值由 Bit 4-7 的 0x3000 决定）
- **Bit 4**: AHT20 传感器就绪标志（0=未接入，1=就绪）
- **Bit 5**: BMP280 传感器就绪标志（0=未接入，1=就绪）
- **Bit 6**: 舵机驱动就绪标志（0=未初始化，1=就绪）
- **Bit 7**: UART 通信正常标志（0=异常，1=正常）
- **Bit 8~15**: 保留（未来扩展）

**心跳寄存器运行机制**：
- **初始值**: `0x3000`（Bit 4-7 固定，Bit 0-3 为 0x0000）
- **翻转逻辑**: 每秒对 Bit 0-3 执行异或操作 `0x3000 ^ 0x000F`
  - 第 1 秒：`0x3000 ^ 0x000F = 0x300F`（心跳位变为 0x000F）
  - 第 2 秒：`0x300F ^ 0x000F = 0x3000`（心跳位恢复 0x0000）
  - 循环往复：`0x3000 ↔ 0x300F`
- **调试优势**: Bit 4-7 固定为 0x3（保留系统状态），Bit 0-3 翻转易于识别存活信号

**舵机速度控制说明**：
- `0~126`：反转调速（0 = 最快反转，126 = 最慢反转）
- `127`：中位停止（死区，舵机静止）
- `128~255`：正转调速（128 = 最慢正转，255 = 最快正转）

## 🚀 启动流程

1. `main()` 完成 `HAL_Init()` 与时钟初始化（72MHz）
2. 初始化 BSP 与协议：`BSP_LED_Init()`、`BSP_UART_Init()`、`Modbus_Init()`
3. `System_Ctrl_Init()` 做配置校验，失败直接 `Error_Handler()` 并记录日志
4. `System_StartTasks()` 创建 LED/UART/Monitor 任务
5. `osKernelStart()` 启动 FreeRTOS 调度器

**任务启动顺序**：
- **Monitor_Task** 优先启动，负责看门狗喂狗（500ms 间隔）与系统心跳维护
- **UART_Task** 次之，处理 Modbus 协议解析与响应
- **LED_Task** 最后启动，提供系统运行状态指示

## 🧪 快速测试

### 测试环境

- **串口助手**：SSCOM 等支持多字符串循环发送的串口工具
- **通信参数**：HEX 模式，波特率 115200，8N1，无校验
- **从机地址**：`1`
- **寄存器映射**：10 个保持寄存器（地址 `0x0000~0x0002` 为传感器数据，`0x0003` 为系统状态，`0x0004` 为 180° 舵机目标角度，`0x0005` 为 360° 舵机速度控制）

### 测试方法：多字符串循环发送

使用串口助手的**多字符串循环发送**功能，按顺序依次发送以下指令（建议设置最后一条指令延时 2500ms，便于观察串口日志输出）：  
- 注意：  
1. 务必勾选 **HEX发送**，也就是直接发送十六进制数据，否则会导致从机无法解析指令，不会产生响应。  
2. 务必加校验位**ModbusCRC16**，串口助手会自动计算并添加校验位，否则会导致从机无响应。  

| 序号 | 发送指令（HEX） | 功能说明 | 延时 (ms) | 预期响应 |
|------|----------------|----------|-----------|----------|
| 1  | ` 01 06 00 04 00 00 C8 0B ` | 写地址 4，角度 0° | 300 | 原样回显 |
| 2  | ` 01 06 00 04 00 5A 48 30 ` | 写地址 4，角度 90° | 300 | 原样回显 |
| 3  | ` 01 06 00 04 00 B4 C8 7C ` | 写地址 4，角度 180° | 300 | 原样回显 |
| 4  | ` 01 06 00 05 00 7F D8 2B ` | 写地址 5，速度 127（停止） | 300 | 原样回显 |
| 5  | ` 01 06 00 05 00 FF D9 8B ` | 写地址 5，速度 255（最大速度顺时针转） | 500 | 原样回显 |
| 6  | ` 01 06 00 05 00 00 99 CB ` | 写地址 5，速度 0（最大速度逆时针转） | 500 | 原样回显 |
| 7  | ` 01 03 00 04 00 01 C5 CB ` | 读地址 4（验证写入值） | 100 | 返回当前角度值 |
| 8  | ` 01 03 00 05 00 01 94 0B ` | 读地址 5（验证写入值） | 100 | 返回当前速度值 |
| 9  | ` 01 03 00 00 00 03 05 CB ` | 读地址 0-2（传感器数据） | 100 | 返回传感器寄存器 |
| 10 | ` 01 03 00 03 00 01 74 0A ` | 读地址 3（系统状态） | 2500 | 返回系统状态 |

**操作步骤**：
1. 在串口助手中切换到"多字符串发送"模式
2. 依次添加上述 10 条指令，并设置对应延时
3. 勾选"循环发送"，点击"发送"开始测试
4. 观察串口接收区：前 9 条指令应正常响应，第 10 条非法帧应无任何响应

### 单指令手动测试

如需单独测试某条指令，可直接在 HEX 发送框输入：  

**读保持寄存器（0x03）**
```
发送: 01 03 00 00 00 03 05 CB   
响应: 返回寄存器地址 0x00、0x01 和 0x02 中的值（温度/湿度/气压）当前未实现  
```

**写单寄存器（0x06）**
```
发送: 01 06 00 04 00 5A 48 30   （把值 90写入地址 4 的寄存器）  
响应: 01 06 00 04 00 5A 48 30   （原样回显）  
现象：180° 舵机角度旋转至 90°  
读取确认: 01 03 00 04 00 01 C5 CB    
响应：01 03 02 00 5A 38 7F     （返回当前寄存器值）  
```

**舵机控制测试**
```
写 180° 舵机（地址 4）:
01 06 00 04 00 00 C8 0B   （舵机转至0°）  
01 06 00 04 00 5A 48 30   （舵机转至90°）  
01 06 00 04 00 B4 C8 7C   （舵机转至180°）  

写 360° 舵机（地址 5）:
01 06 00 05 00 00 99 CB   （舵机最大速度逆时针转）  
01 06 00 05 00 7F D8 2B   （停止）  
01 06 00 05 00 FF D9 8B   （舵机最大速度顺时针转）  
```

**读系统状态（地址 3）**  
```
发送: 01 03 00 03 00 01 74 0A 
```
心跳寄存器响应：  
当第 0 秒读取响应：`01 03 02 30 00 AC 44 `（心跳位 0x3000，1 秒后翻转）  
第 1 秒读取：`01 03 02 30 0F EC 40 `（心跳位 0x300F）  
第 2 秒读取：`01 03 02 30 00 AC 00`（心跳位恢复 0x3000）  
- **规律**: `0x3000 ↔ 0x300F` 每秒交替，证明系统存活  

### 异常情况测试

- **CRC 错帧**：无任何响应（直接丢弃，如测试表第 10 条）
- **写越界地址**：
 `01 06 00 10 00 01  `
 返回 `01 86 02 C3 A1`（异常码 0x02）
- **非法值**：返回 `01 86 03 XX XX`（异常码 0x03）
- **写入只读寄存器**：返回 `01 86 03 XX XX`（异常码 0x03）

## 🔍 调试与排障要点

### 错误日志格式

```
ERR[类型] 文件:行号
```

类型编码：

- `ERR[1]`: ERROR\_HAL（HAL 调用或底层初始化错误）
- `ERR[2]`: ERROR\_UART（UART 接收链 / 恢复链异常）
- `ERR[4]`: ERROR\_STACK\_OVERFLOW（FreeRTOS 栈溢出）
- `ERR[5]`: ERROR\_SYSTEM（系统控制层错误）
- `ERR[6-9]`: 硬件故障（HardFault/MemManage/BusFault/UsageFault）

### 常见问题排查

1. **LED 常亮且无任务输出**：优先检查是否进入 `System_EnterSafeMode()`
2. **ERR\[2] 高频刷屏**：检查 UART 恢复路径是否误判 `HAL_BUSY`
3. **Modbus 无响应**：
   - 确认 HEX 模式、关闭自动换行
   - 检查主从波特率一致
   - 验证 CRC 校验码正确性
4. **图片不显示**：确认文件已放入 `docs/images/` 且文件名与文档中一致

## ⚠️ 已知约束与局限性

### 功能限制

- 当前仅实现 `0x03`（读保持寄存器）和 `0x06`（写单寄存器）功能码
- 寄存器总数：10 个，地址 `0x0004`（舵机角度）和 `0x0005`（舵机速度）可写，地址 `0x0000~0x0003` 为只读传感器/系统数据
- 未实现写多个寄存器（0x10）、线圈/离散输入等功能码
- 仅支持单字节 Modbus 从机地址（当前固定为 0x01）

### 测试局限性

- **UART 硬件错误触发**：实验室环境无法可靠触发 ORE/FE/NE 硬件错误标志位，真实工业干扰场景需进一步验证
- **长稳测试**：建议进行 24h+ 连续运行测试，验证任务栈稳定性与内存泄漏情况
- **舵机控制测试**：当前已验证 180° 舵机角度控制和 360° 舵机速度控制功能，但长期机械磨损影响未评估

### 硬件建议

- 工业现场建议增加板级外部上拉/终端电阻，`GPIO_PULLUP` 仅作辅助
- RS485 A/B 线建议使用双绞线，降低电磁干扰影响
- 舵机供电建议独立于 MCU 电源，避免 PWM 切换时的电源波动干扰
