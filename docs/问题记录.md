# Modbus RTU网关项目 - 问题清单与改进计划

**文档版本**：V1.9  
**梳理日期**：2026-05-20  
**适用范围**：STM32F103 + FreeRTOS Modbus RTU网关项目

**优先级定义**
- P0：阻断性问题，立即处理（24小时内）
- P1：高风险问题，尽快处理（3天内）
- P2：优化类问题，中期迭代改进（一周内）
- P3：低优先级问题，长期规划改进

**归档说明**：
- 已验证通过的问题已移至 [docs/ignore_docs/已解决问题清单.md](docs/ignore_docs/已解决问题清单.md)

## 一、P1 级 高风险待优化问题（尽快处理）

### P1-003 系统缺少资源泄漏与运行监控机制
涉及模块：全局内存、任务句柄、串口缓冲区  
问题描述：无内存占用统计、无缓冲区使用率监控、无资源申请释放校验。  
核心风险：长期长时间运行易出现内存溢出、缓冲区溢出、隐性资源耗尽，引发偶发死机。  
改进建议：  
1. 基于 FreeRTOS 增加任务栈水位常驻监控；  
2. 统计各静态缓冲区实时占用情况；  
3. 对信号量、互斥锁等内核对象增加创建/释放匹配校验。  
当前状态：未处理

### P1-005 串口中断优先级配置合理性待评估
涉及模块：Bsp/Src/uart.c 中断初始化配置  
问题描述：当前串口中断优先级固定配置为5，未结合整体任务优先级矩阵统筹设计。  
核心风险：可能抢占系统高优先级关键任务，影响系统实时性与稳定性。  
改进建议：  
1. 梳理全系统中断、任务优先级排布表；  
2. 调整串口中断优先级为适中层级，不抢占核心系统任务；  
3. 调整后长时间压测验证通信实时性无退化。  
当前状态：未处理

### P1-006 CRC16 采用逐字节循环计算，性能偏低
涉及模块：App/Protocol/Src/modbus.c  
问题描述：Modbus CRC16 采用实时逐字节移位计算方式，运算效率一般。  
核心风险：高频连续轮询、多帧并发场景下，增加协议处理耗时，挤占任务调度时间片。  
改进建议：  
1. 改用 CRC16-Modbus 预查表法替代实时循环计算；  
2. 固化 CRC 常量表，运行时直接索引取值；  
3. 对比优化前后耗时，保证高负载下协议处理无卡顿。  
当前状态：未处理

### P1-008 UART 驱动互斥/阻塞使用风险
涉及模块：`App/Driver/Src/driver_uart.c`、`Bsp/Src/uart.c`
问题描述：`UART_Driver_Send` 持有互斥量期间调用阻塞式 `BSP_UART_Send`，且超时值依赖传入参数，配置不当易导致发送失败或频繁重试。
核心风险：长时阻塞占用互斥量影响其他发送路径；若在中断上下文误调用将引发不可预期行为。
改进建议：1. 注释强制声明仅限任务上下文调用；2. 定义明确互斥超时常量；3. 长期改为非阻塞环形队列+发送任务机制。
当前状态：未处理

### P1-009 UART 接收轮询与延迟窗口
涉及模块：`App/Driver/Src/driver_uart.c`、`App/Task/Src/uart_task.c`
问题描述：`UART_Driver_Receive` 及 `uart_task` 采用 `osDelay(1)` 轮询模型，引入最大 1ms 延迟窗口，高频场景下增加 CPU 空转。
核心风险：高并发或低间隔请求场景下，轮询延迟影响实时性并增加响应滞后。
改进建议：1. 使用中断通知（Semaphore/Notify）替代轮询；2. 短期需文档化延迟预期并实测分布。
当前状态：未处理

## 二、P2 级 中期架构与可维护性优化（迭代改进）

### P2-001 modbus.c 单体文件过大，模块耦合偏高
涉及模块：App/Protocol 协议层  
问题描述：协议解析、寄存器映射、异常响应全部堆砌在单个文件，职责不拆分。  
核心风险：可读性差、耦合严重、新增功能易牵一发而动全身，不利于后期扩展主机、多功能码。  
改进建议：  
1. 模块化拆分：modbus_core 协议帧处理、modbus_map 寄存器映射、modbus_excp 异常响应；  
2. 严格分层：驱动层只提供收发硬件接口，协议层不直接操作外设句柄。  
当前状态：部分处理

### P2-003 缺少单元测试、静态检查与自动化流程
涉及模块：工程整体开发流程  
问题描述：无核心算法单元测试、无静态代码检查、无编译以外的质量管控。  
核心风险：功能迭代后回归测试靠人工，容易隐性引入 bug，版本质量不可控。  
改进建议：  
1. 对 CRC 计算、Modbus 帧解析、异常响应做简易单元测试；  
2. 引入静态语法检查工具做代码规范校验；  
3. 建立基础编译+静态检查的简易版本发布流程。  
当前状态：未处理

### P2-005 系统缺少内存、栈、缓冲区资源运行时监控
涉及模块：系统监控任务、全局资源  
问题描述：仅基础栈水位打印，无整体资源占用、缓冲区负载、内存趋势监控。  
核心风险：长期运行无法提前预判栈溢出、缓冲区爆满、内存持续增长等隐患。  
改进建议：  
1. 在监控任务中常驻统计各任务栈剩余、缓冲区使用率；  
2. 资源临近阈值主动打印告警日志；  
3. 留存运行资源快照，便于后期问题复盘。  
当前状态：未处理
