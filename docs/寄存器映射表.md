## ⚙️ 寄存器默认初始值

**代码位置**：[`modbus.c:25-26`](../App/Protocol/Src/modbus.c#L25-L26)

``` c
//（舵机寄存器默认值超出合法范围, 强制主机首次写入合法角度后舵机才能使用）

static const uint16_t s_default_regs[] = {
    0x0000U,  // 对应地址 0x0000: AHT20 温度（预留）
    0x1000U,  // 对应地址 0x0001: AHT20 湿度（预留）
    0x2000U,  // 对应地址 0x0002: BMP280 气压（预留）
    0x3000U,  // 对应地址 0x0003: 系统状态寄存器（心跳+标志位）
    0x4000U,   // 对应地址 0x0004: 180° 舵机目标角度
    0x5000U   // 对应地址 0x0005: 360° 舵机速度/方向
};
//剩余地址0x0007-0x0009寄存器值默认为0x0000
```

### 默认值设计说明

| 寄存器地址 | 默认值（十六进制） | 默认值（十进制） | 设计意图 |
|-----------|------------------|----------------|---------|
| 0x0000 | `0x0000` | 0 | 传感器未接入时返回零值，便于主机判断设备状态 |
| 0x0001 | `0x1000` | 4096 | 占位符，区分于温度寄存器（0x0000），避免误判 |
| 0x0002 | `0x2000` | 8192 | 占位符，区分于温湿度寄存器，便于调试识别 |
| 0x0003 | `0x3000` | 12288 | Bit 8-15 = 0x30（系统标识），Bit 0-7 = 0x00（初始心跳位） |
| 0x0004 | `0x4000` | 16384 | **故意设置为非法值**，强制主机首次写入合法角度（0-180）后才可使用舵机 |
| 0x0005 | `0x0000` | 0 | 360° 舵机默认反转最快，需主机显式写入中位（0x7F）停止 |

**⚠️ 重要提示**：
- **不要随意修改默认值**：这些值经过精心设计，用于区分不同寄存器类型和强制主机初始化
- **地址 0x0004 的非法默认值**：确保舵机在上电后不会意外转动，必须由主机显式写入合法角度
- **传感器占位符**：0x1000/0x2000 用于调试时快速识别寄存器类型，实际应用中应替换为真实传感器数据

## 🔍 寄存器详细说明

### 0x0003 - 系统状态寄存器

**读写权限**：只读（R）  
**初始值**：`0x3000`（上电复位后）  
**更新频率**：每秒自动更新（心跳 Bit 0~3）

#### 位定义

| Bit 位 | 名称 | 含义 | 默认值 | 说明 |
|--------|------|------|--------|------|
| 0~3 | `HEARTBEAT_COUNTER` | 心跳计数器 | 动态翻转 | 0x0000 ↔ 0x000F，每秒翻转一次 |
| 4 | `AHT20_READY` | AHT20 传感器就绪标志 | 0 | 0=未接入，1=就绪（待实现） |
| 5 | `BMP280_READY` | BMP280 传感器就绪标志 | 0 | 0=未接入，1=就绪（待实现） |
| 6 | `SERVO_DRIVER_READY` | 舵机驱动就绪标志 | 1 | 0=未初始化，1=就绪（待实现） |
| 7 | `UART_COMM_NORMAL` | UART 通信正常标志 | 1 | 0=异常，1=正常（待实现） |
| 8~15 | `RESERVED` | 保留 | 0x30 | 固定为 0x30（系统标识） |

#### 心跳运行机制

- **初始值**: `0x3000`（Bit 4-7 = 0x3，Bit 0-3 = 0x0）
- **翻转逻辑**: 每秒执行 `current_val ^ 0x000F`
  - 第 0 秒：`0x3000`（心跳位 0x0000）
  - 第 1 秒：`0x300F`（心跳位 0x000F）
  - 第 2 秒：`0x3000`（心跳位恢复 0x0000）
  - 循环往复：`0x3000 ↔ 0x300F`

#### 读取示例

**发送指令**：`01 03 00 03 00 01 74 0A`

**预期响应**（第 0 秒）：`01 03 02 30 00 AC 44`  
**预期响应**（第 1 秒）：`01 03 02 30 0F EC 40`

**验证方法**：
连续读取该寄存器，观察高字节的变化：
```
时间 T0: 01 03 02 30 00 AC 44  （0x3000）
时间 T1: 01 03 02 30 0F EC 40  （0x300F）
时间 T2: 01 03 02 30 00 AC 44  （0x3000）
...
```

### 0x0004 - 180° 舵机目标角度

**读写权限**：可写（W）  
**控制方式**：Modbus 写单寄存器（0x06）

#### 参数映射

| 写入值（十六进制） | 对应角度 | PWM 脉宽 | 占空比 |
|--------|----------|----------|--------|
| 0x00 |   0° | 0.5ms |  2.5% |
| 0x5A |  90° | 1.5ms |  7.5% |
| 0xB4 | 180° | 2.5ms | 12.5% |

#### 计算公式

```c
// 角度 → PWM 脉宽（微秒）
pulse_width_us = 500 + (angle * 2000 / 180);

// PWM 脉宽 → 占空比（50Hz，周期 20ms）
duty_cycle = pulse_width_us / 20000.0;
```

#### 写入示例

**发送指令**：`01 06 00 04 00 5A 48 30`（写入 90°）

**预期响应**：`01 06 00 04 00 5A 48 30`（原样回显）

**验证方法**：
1. 发送写入指令
2. 观察舵机是否转到 90°
3. 发送读取指令 `01 03 00 04 00 01 C5 CB` 确认返回值

**读取确认响应**：`01 03 02 00 5A 38 7F`

#### 错误处理

| 错误场景 | 返回异常帧 | 异常码 |
|----------|-----------|--------|
| 写入值 > 0xB4 | `01 86 03 02 61 ` | 0x03（非法数据值） |
| 写入值 < 0x00 | `01 86 03 02 61 ` | 0x03（非法数据值） |

### 0x0005 - 360° 舵机速度/方向

**读写权限**：可写（W）  
**控制方式**：Modbus 写单寄存器（0x06）

#### 参数映射

| 写入值 | 运动状态 | PWM 脉宽 | 占空比 | 说明 |
|--------|----------|----------|--------|------|
| 0x00~0x75 | 反转调速 | 2.5~1.58ms | 12.5~7.9% | 值越小反转越快（有效下限117） |
| **0x76~0x88** | **死区保护** | **1.5ms** | **7.5%** | **固件强制归零至0x7F，舵机静止** |
| 0x89~0xFF | 正转调速 | 1.42~0.5ms | 7.1~2.5% | 值越大正转越快（有效上限137） |

**⚠️ 死区保护机制**：
- **自动过滤范围**：0x76-0x88（十进制118-136，共19个值，包含边界值0x76和0x88）
- **保护策略**：写入此范围内的任意值，固件内部自动转换为0x7F（完全停止）
- **设计目的**：避免PWM脉宽变化过小（<80μs）导致舵机在"微动"与"静止"间随机切换，应对长时间运转后的机械磨损和温度漂移

**✅ 固件已实现自动死区保护**：
- **死区范围**：0x76-0x88（±10档，对应PWM变化约±80μs）
- **保护策略**：写入此范围内的任意值，固件自动强制归零至0x7F（完全停止）
- **有效速度边界**：
  - 反转有效范围：0x00-0x75（0x75对应慢速反转，0x00对应最大速度反转）  
  - 正转有效范围：0x88-0xFF（0x88对应慢速正转，0xFF对应最大速度正转）   

**建议**：
1. **跳变策略**：如需反转→正转切换，建议直接跳变至目标速度（如0x60→0xA0），避免逐步穿越死区
2. **最小有效速度**：实际应用中建议设置最小有效速度为 0x50（反转）和 0xA0（正转），预留更大安全裕度
3. **应用层滤波**：如需更平滑控制，可在上位机实现滞回比较器或斜坡函数

#### 写入示例

**正转**：
- 发送指令：`01 06 00 05 00 FF D9 8B`（写入 255）
- 预期响应：`01 06 00 05 00 FF D9 8B`
- 观察现象：舵机正转最快

**停止**：
- 发送指令：`01 06 00 05 00 7F D8 2B`（写入 127）
- 预期响应：`01 06 00 05 00 7F D8 2B`
- 观察现象：舵机完全静止

**反转**：
- 发送指令：`01 06 00 05 00 00 99 CB`（写入 0）
- 预期响应：`01 06 00 05 00 00 99 CB`
- 观察现象：舵机反转最快

#### 错误处理

| 错误场景 | 返回异常帧 | 异常码 |
|----------|-----------|--------|
| 写入值 > 0xFF  | `01 86 03 02 61 ` | 0x03（非法数据值） |
| 写入值 < 0x00 | `01 86 03 02 61 ` | 0x03（非法数据值） |

## 🛠️ 常用 Modbus 指令速查

### 读保持寄存器（0x03）

**格式**：`[从机地址] [功能码] [起始地址高] [起始地址低] [数量高] [数量低] [CRC 低] [CRC 高]`

**示例**：
```
# 读地址 0x0004（180° 舵机当前角度）
01 03 00 04 00 01 C5 CB

# 读地址 0x0000~0x0002（传感器数据）
01 03 00 00 00 03 05 CB

# 读地址 0x0003（系统状态）
01 03 00 03 00 01 74 0A
```

### 写单寄存器（0x06）

**格式**：`[从机地址] [功能码] [地址高] [地址低] [值高] [值低] [CRC 低] [CRC 高]`

**示例**：
```
# 写地址 0x0004，值 90（舵机转到 90°）
01 06 00 04 00 5A 48 30

# 写地址 0x0005，值 255（360° 舵机正转最快）
01 06 00 05 00 FF D9 8B
```

### 异常响应

**格式**：`[从机地址] [功能码+0x80] [异常码] [CRC 低] [CRC 高]`

**异常码定义**：
- `0x01`：非法功能码
- `0x02`：非法数据地址
- `0x03`：非法数据值

**示例**：
```
# 越界地址访问（地址 0x0010 超出范围）
请求: 01 06 00 10 00 01 49 CF
响应: 01 86 02 C3 A1  （异常码 0x02）

# 非法值写入（值 500 超出 0~180 范围）
请求: 01 06 00 04 01 F4 C8 1C 
响应: 01 86 03 02 61   （异常码 0x03）
```

## ⚠️ 注意事项

### 1. CRC 校验
- **字节序**：低字节在前，高字节在后
- **算法**：CRC-16/MODBUS（多项式 0x8005）
- **工具推荐**：串口助手内置 ModbusCRC16 自动计算功能（务必勾选）

### 2. 响应超时
- **典型延迟**：< 1ms
- **超时阈值**：建议设置 100ms（包含网络抖动余量）

### 3. 并发访问
- 寄存器访问已加互斥锁保护
- 单次加锁快照读取，避免并发一致性风险
- 写回调锁外执行，避免阻塞主流程

### 4. 断电保存
- 当前版本不支持寄存器值断电保存
- 重启后所有可写寄存器恢复默认值
