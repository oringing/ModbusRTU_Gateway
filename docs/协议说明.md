# Modbus RTU 协议实现说明

## 1. 协议实现范围

### 1.1 基本配置
- **工作模式**：Modbus RTU 从机
- **从机地址**：`MODBUS_SLAVE_ADDR = 0x01`
- **校验方式**：Modbus CRC-16（低字节在前，高字节在后）

### 1.2 已实现功能码
| 功能码 | 功能描述 | 状态 |
|--------|---------|------|
| `0x03` | 读保持寄存器 | ✅ 稳定实现 |
| `0x06` | 写单个寄存器 | ✅ 稳定实现 |

---

## 2. 帧处理流程

### 2.1 接收处理链路
```
UART_Driver_Receive() 
  → Modbus_ValidateFrame()  // 帧合法性校验
    → Modbus_Process()      // 功能码分发
      → 0x03/0x06 处理函数
```

### 2.2 帧校验规则（Modbus_ValidateFrame）
1. **最小帧长校验**：`frame_len ≥ MODBUS_MIN_FRAME_LEN (8 bytes)`（0x03/0x06 请求帧固定长度）
2. **从机地址校验**：`frame[MODBUS_RESP_ADDR_IDX] == MODBUS_SLAVE_ADDR`
3. **CRC-16 校验**：
   - **字节序**：低字节在前（`frame[len-2]`），高字节在后（`frame[len-1]`）
   - **校验范围**：除 CRC 外的所有字节（`frame[0] ~ frame[len-3]`）
   - **算法**：标准 Modbus CRC-16（多项式 `0xA001`，初始值 `0xFFFF`）

## 3. 功能码 `0x03` 实现规范

### 3.1 请求帧格式
| 字节偏移 | 字段 | 长度 | 宏定义 | 说明 |
|---------|------|------|--------|------|
| 0 | 从机地址 | 1 byte | `MODBUS_RESP_ADDR_IDX` | `0x01` |
| 1 | 功能码 | 1 byte | `MODBUS_RESP_FUNC_CODE_IDX` | `0x03` |
| 2-3 | 起始地址 | 2 bytes | `MODBUS_REQ_START_ADDR_HIGH_IDX` / `LOW_IDX` | 高字节在前（Big-Endian） |
| 4-5 | 寄存器数量 | 2 bytes | `MODBUS_REQ_REG_COUNT_HIGH_IDX` / `LOW_IDX` | 高字节在前 |
| 6-7 | CRC-16 | 2 bytes | - | **低字节在前，高字节在后** |

**示例帧**：`01 03 00 00 00 05 [CRC_L] [CRC_H]`
- 读取从地址 0 开始的 5 个寄存器

### 3.2 响应帧格式
| 字节偏移 | 字段 | 长度 | 宏定义 | 说明 |
|---------|------|------|--------|------|
| 0 | 从机地址 | 1 byte | `MODBUS_RESP_ADDR_IDX` | `0x01` |
| 1 | 功能码 | 1 byte | `MODBUS_RESP_FUNC_CODE_IDX` | `0x03` |
| 2 | 字节计数 | 1 byte | `MODBUS_RESP_BYTE_COUNT_IDX` | `寄存器数量 × 2` |
| 3~N | 寄存器数据 | N bytes | `MODBUS_RESP_DATA_START_IDX` + offset | 每个寄存器 2 bytes（高字节在前） |
| N+1~N+2 | CRC-16 | 2 bytes | - | **低字节在前，高字节在后** |

**示例响应**：`01 03 0A [Reg0_H] [Reg0_L] ... [Reg4_H] [Reg4_L] [CRC_L] [CRC_H]`

### 3.3 入参校验规则
| 校验项 | 规则 | 异常码 |
|--------|------|--------|
| 起始地址 | `start_addr < MODBUS_REG_MAX_COUNT` | `0x02`（非法数据地址） |
| 寄存器数量 | `reg_count ≥ 1 && reg_count ≤ MODBUS_MAX_READ_REGS (10)` | `0x03`（非法数据值） |
| 地址+数量边界 | `start_addr + reg_count ≤ MODBUS_REG_MAX_COUNT` | `0x02` |

## 4. 功能码 `0x06` 实现规范

### 4.1 请求帧格式
| 字节偏移 | 字段 | 长度 | 宏定义 | 说明 |
|---------|------|------|--------|------|
| 0 | 从机地址 | 1 byte | `MODBUS_RESP_ADDR_IDX` | `0x01` |
| 1 | 功能码 | 1 byte | `MODBUS_RESP_FUNC_CODE_IDX` | `0x06` |
| 2-3 | 寄存器地址 | 2 bytes | `MODBUS_REQ_REG_ADDR_HIGH_IDX` / `LOW_IDX` | 高字节在前 |
| 4-5 | 寄存器值 | 2 bytes | `MODBUS_REQ_REG_VALUE_HIGH_IDX` / `LOW_IDX` | 高字节在前 |
| 6-7 | CRC-16 | 2 bytes | - | **低字节在前，高字节在后** |

**示例帧**：`01 06 00 04 00 5A [CRC_L] [CRC_H]`
- 将地址 0x0004 的寄存器写入值 0x005A（十进制 90）

### 4.2 成功响应
- **响应内容**：原样回显请求帧（8 bytes）
- **响应时机**：寄存器值写入成功后立即回显

### 4.3 入参校验规则
| 校验项 | 规则 | 异常码 |
|--------|------|--------|
| 寄存器地址 | `addr == MODBUS_REG_ADDR_SERVO_TARGET` 或 `MODBUS_REG_ADDR_SERVO_SPEED` | `0x02` |
| 只读标志 | `holding_regs[addr].read_only == false` | `0x03` |
| 写入值范围 | `0x0004: 0 ~ SERVO_DRIVER_TARGET_MAX (180)`（舵机角度）<br>`0x0005: 0 ~ SERVO_DRIVER_SPEED_MAX (255)`（舵机速度） | `0x03` |

---

## 5. 异常响应规范

### 5.1 异常帧格式
| 字节偏移 | 字段 | 长度 | 宏定义 | 说明 |
|---------|------|------|--------|------|
| 0 | 从机地址 | 1 byte | `MODBUS_EX_RESP_ADDR_IDX` | `0x01` |
| 1 | 异常功能码 | 1 byte | `MODBUS_EX_RESP_FUNC_CODE_IDX` | `原功能码 | 0x80` |
| 2 | 异常码 | 1 byte | `MODBUS_EX_RESP_EX_CODE_IDX` | 见下表 |
| 3-4 | CRC-16 | 2 bytes | - | **低字节在前，高字节在后** |

### 5.2 异常码定义
| 异常码 | 含义 | 触发条件 |
|--------|------|---------|
| `0x01` | 非法功能码 | 请求功能码非 0x03/0x06 |
| `0x02` | 非法数据地址 | 寄存器地址越界或不允许写入 |
| `0x03` | 非法数据值 | 寄存器数量非法或写入值超限 |

### 5.3 静默丢弃场景（不响应）
- CRC 校验失败
- 从机地址不匹配
- 帧长度不符合最小要求（< `MODBUS_MIN_FRAME_LEN`）

## 6. 寄存器映射表

**详细说明**：完整的寄存器地址表、默认值设计、死区保护机制请参见 [寄存器映射表.md](寄存器映射表.md)

### 6.1 寄存器访问策略
- **读操作（0x03）**：单次加锁快照读取，避免并发不一致
- **写操作（0x06）**：锁内更新值，锁外执行回调（避免死锁）
- **互斥保护**：所有寄存器访问通过 `s_modbus_reg_mutex` 互斥锁保护
