# Modbus RTU 协议实现说明

## 1. 协议实现范围

### 1.1 基本配置
- **工作模式**：Modbus RTU 从机
- **从机地址**：`MODBUS_SLAVE_ADDR = 0x01`
- **校验方式**：Modbus CRC-16（低字节在前，高字节在后）

### 1.2 已实现功能码
| 功能码 | 功能描述 | 状态 |
|--------|---------|------|
| `0x03` | 读保持寄存器 | ✅ 稳定实现 |
| `0x06` | 写单个寄存器 | ✅ 稳定实现 |

---

## 2. 帧处理流程

### 2.1 接收处理链路
```
UART_Driver_Receive() 
  → Modbus_ValidateFrame()  // 帧合法性校验
    → Modbus_Process()      // 功能码分发
      → 0x03/0x06 处理函数
```

### 2.2 帧校验规则（Modbus_ValidateFrame）
1. **最小帧长校验**：`frame_len ≥ MODBUS_MIN_FRAME_LEN (8 bytes)`（0x03/0x06 请求帧固定长度）
2. **从机地址校验**：`frame[MODBUS_RESP_ADDR_IDX] == MODBUS_SLAVE_ADDR`
3. **CRC-16 校验**：
   - **字节序**：低字节在前（`frame[len-2]`），高字节在后（`frame[len-1]`）
   - **校验范围**：除 CRC 外的所有字节（`frame[0] ~ frame[len-3]`）
   - **算法**：标准 Modbus CRC-16（多项式 `0xA001`，初始值 `0xFFFF`）

## 3. 功能码 `0x03` 实现规范

### 3.1 请求帧格式
| 字节偏移 | 字段 | 长度 | 宏定义 | 说明 |
|---------|------|------|--------|------|
| 0 | 从机地址 | 1 byte | `MODBUS_RESP_ADDR_IDX` | `0x01` |
| 1 | 功能码 | 1 byte | `MODBUS_RESP_FUNC_CODE_IDX` | `0x03` |
| 2-3 | 起始地址 | 2 bytes | `MODBUS_REQ_START_ADDR_HIGH_IDX` / `LOW_IDX` | 高字节在前（Big-Endian） |
| 4-5 | 寄存器数量 | 2 bytes | `MODBUS_REQ_REG_COUNT_HIGH_IDX` / `LOW_IDX` | 高字节在前 |
| 6-7 | CRC-16 | 2 bytes | - | **低字节在前，高字节在后** |

**示例帧**：`01 03 00 00 00 05 [CRC_L] [CRC_H]`
- 读取从地址 0 开始的 5 个寄存器

### 3.2 响应帧格式
| 字节偏移 | 字段 | 长度 | 宏定义 | 说明 |
|---------|------|------|--------|------|
| 0 | 从机地址 | 1 byte | `MODBUS_RESP_ADDR_IDX` | `0x01` |
| 1 | 功能码 | 1 byte | `MODBUS_RESP_FUNC_CODE_IDX` | `0x03` |
| 2 | 字节计数 | 1 byte | `MODBUS_RESP_BYTE_COUNT_IDX` | `寄存器数量 × 2` |
| 3~N | 寄存器数据 | N bytes | `MODBUS_RESP_DATA_START_IDX` + offset | 每个寄存器 2 bytes（高字节在前） |
| N+1~N+2 | CRC-16 | 2 bytes | - | **低字节在前，高字节在后** |

**示例响应**：`01 03 0A [Reg0_H] [Reg0_L] ... [Reg4_H] [Reg4_L] [CRC_L] [CRC_H]`

### 3.3 入参校验规则
| 校验项 | 规则 | 异常码 |
|--------|------|--------|
| 起始地址 | `start_addr < MODBUS_REG_MAX_COUNT` | `0x02`（非法数据地址） |
| 寄存器数量 | `reg_count ≥ 1 && reg_count ≤ MODBUS_MAX_READ_REGS (10)` | `0x03`（非法数据值） |
| 地址+数量边界 | `start_addr + reg_count ≤ MODBUS_REG_MAX_COUNT` | `0x02` |

## 4. 功能码 `0x06` 实现规范

### 4.1 请求帧格式
| 字节偏移 | 字段 | 长度 | 宏定义 | 说明 |
|---------|------|------|--------|------|
| 0 | 从机地址 | 1 byte | `MODBUS_RESP_ADDR_IDX` | `0x01` |
| 1 | 功能码 | 1 byte | `MODBUS_RESP_FUNC_CODE_IDX` | `0x06` |
| 2-3 | 寄存器地址 | 2 bytes | `MODBUS_REQ_REG_ADDR_HIGH_IDX` / `LOW_IDX` | 高字节在前 |
| 4-5 | 寄存器值 | 2 bytes | `MODBUS_REQ_REG_VALUE_HIGH_IDX` / `LOW_IDX` | 高字节在前 |
| 6-7 | CRC-16 | 2 bytes | - | **低字节在前，高字节在后** |

**示例帧**：`01 06 00 04 00 5A [CRC_L] [CRC_H]`
- 将地址 0x0004 的寄存器写入值 0x005A（十进制 90）

### 4.2 成功响应
- **响应内容**：原样回显请求帧（8 bytes）
- **响应时机**：寄存器值写入成功后立即回显

### 4.3 入参校验规则
| 校验项 | 规则 | 异常码 |
|--------|------|--------|
| 寄存器地址 | `addr == MODBUS_REG_ADDR_SERVO_TARGET` 或 `MODBUS_REG_ADDR_SERVO_SPEED` | `0x02` |
| 只读标志 | `holding_regs[addr].read_only == false` | `0x03` |
| 写入值范围 | `0x0004: 0 ~ SERVO_DRIVER_TARGET_MAX (180)`（舵机角度）<br>`0x0005: 0 ~ SERVO_DRIVER_SPEED_MAX (255)`（舵机速度） | `0x03` |

---

## 5. 异常响应规范

### 5.1 异常帧格式
| 字节偏移 | 字段 | 长度 | 宏定义 | 说明 |
|---------|------|------|--------|------|
| 0 | 从机地址 | 1 byte | `MODBUS_EX_RESP_ADDR_IDX` | `0x01` |
| 1 | 异常功能码 | 1 byte | `MODBUS_EX_RESP_FUNC_CODE_IDX` | `原功能码 | 0x80` |
| 2 | 异常码 | 1 byte | `MODBUS_EX_RESP_EX_CODE_IDX` | 见下表 |
| 3-4 | CRC-16 | 2 bytes | - | **低字节在前，高字节在后** |

### 5.2 异常码定义
| 异常码 | 含义 | 触发条件 |
|--------|------|---------|
| `0x01` | 非法功能码 | 请求功能码非 0x03/0x06 |
| `0x02` | 非法数据地址 | 寄存器地址越界或不允许写入 |
| `0x03` | 非法数据值 | 寄存器数量非法或写入值超限 |

### 5.3 静默丢弃场景（不响应）
- CRC 校验失败
- 从机地址不匹配
- 帧长度不符合最小要求（< `MODBUS_MIN_FRAME_LEN`）

## 6. 寄存器映射表

### 6.1 寄存器配置
| 地址 | 宏定义 | 功能描述 | 默认值 | 读写属性 | 合法范围 | 备注 |
|------|--------|----------|:------:|-------:|:--------:|:--------:|------|
| `0x0000` | `MODBUS_REG_ADDR_SENSOR_TEMP` | 温度数据 | `0x1234` | 只读 | - | 占位符（待接入 DHT11） |
| `0x0001` | `MODBUS_REG_ADDR_SENSOR_HUMIDITY` | 湿度数据 | `0x5678` | 只读 | - | 占位符（待接入 DHT11） |
| `0x0002` | `MODBUS_REG_ADDR_DEVICE_STATUS` | 设备状态字 | `0xABCD` | 只读 | - | 心跳/系统状态 |
| `0x0003` | `MODBUS_REG_ADDR_SYSTEM_FLAGS` | 系统配置标志 | `0x1234` | 只读 | - | 预留 |
| `0x0004` | `MODBUS_REG_ADDR_SERVO_TARGET` | 180° 舵机目标角度 | `0x005A` (90) | **可写** | `0 ~ SERVO_DRIVER_TARGET_MAX (180)` | 映射至 PA6 (TIM3_CH1) |
| `0x0005` | `MODBUS_REG_ADDR_SERVO_SPEED` | 360° 舵机速度/方向 | `0x007F` (127) | **可写** | `0 ~ SERVO_DRIVER_SPEED_MAX (255)` | 映射至 PA7 (TIM3_CH2) |
| `0x0006 ~ 0x0009` | `MODBUS_REG_ADDR_RESERVED_BEGIN ~ END` | 预留扩展 | `0x0000` | - | - | 未来功能预留 |

**寄存器总数**：`MODBUS_REG_MAX_COUNT = 10`

### 6.2 寄存器访问策略
- **读操作（0x03）**：单次加锁快照读取，避免并发不一致
- **写操作（0x06）**：锁内更新值，锁外执行回调（避免死锁）
- **互斥保护**：所有寄存器访问通过 `s_modbus_reg_mutex` 互斥锁保护

## 7. 寄存器地址映射

### 7.1 保持寄存器地址表（10 个）

| 地址 | 宏定义 | 功能说明 | 读写 | 范围 | 联动功能 |
|------|--------|----------|------|------|----------|
| 0x0000 | `MODBUS_REG_ADDR_SENSOR_TEMP` | AHT20 温度（预留） | R | 0x1234 | 待接入 AHT20 |
| 0x0001 | `MODBUS_REG_ADDR_SENSOR_HUMIDITY` | AHT20 湿度（预留） | R | 0x5678 | 待接入 AHT20 |
| 0x0002 | `MODBUS_REG_ADDR_SENSOR_PRESSURE` | BMP280 气压（预留） | R | 0x2710 | 待接入 BMP280 |
| 0x0003 | `MODBUS_REG_ADDR_DEVICE_STATUS` | **系统状态寄存器** | R | 位定义见下 | 心跳+故障标志 |
| 0x0004 | `MODBUS_REG_ADDR_SERVO_TARGET` | **180° 舵机目标角度** | **W** | 0~180 | TIM3_CH1/PA6 |
| 0x0005 | `MODBUS_REG_ADDR_SERVO_SPEED` | **360° 舵机速度/方向** | **W** | 0~255 | TIM3_CH2/PA7 |
| 0x0006~0x0009 | - | 预留扩展 | R | 0x0000 | 未来功能扩展 |

**0x0003 系统状态寄存器位定义**：
- **Bit 0~3**: 心跳计数器（0x000F ↔ 0x0000，每秒翻转）
- **Bit 4**: AHT20 传感器就绪标志（0=未接入，1=就绪）
- **Bit 5**: BMP280 传感器就绪标志（0=未接入，1=就绪）
- **Bit 6**: 舵机驱动就绪标志（0=未初始化，1=就绪）
- **Bit 7**: UART 通信正常标志（0=异常，1=正常）
- **Bit 8~15**: 保留（未来扩展）

**舵机速度控制说明**：
- `0~126`：反转调速（0 = 最快反转，126 = 最慢反转）
- `127`：中位停止（死区，舵机静止）
- `128~255`：正转调速（128 = 最慢正转，255 = 最快正转）

## 8. CRC-16 校验实现细节

### 8.1 字节序说明
**重要**：Modbus RTU 帧中的 CRC 字节序与常规 Big-Endian 不同：
- **帧中存储顺序**：`[...数据..., CRC_Low_Byte, CRC_High_Byte]`
- **重组为 16 位值**：`CRC_Value = (CRC_High << 8) | CRC_Low`

**示例**：
- 计算得到的 CRC 值：`0x1234`
- 帧中存储：`[...数据..., 0x34, 0x12]`（低字节 `0x34` 在前）

### 8.2 算法参数
- **多项式**：`0xA001`（反转的 `0x8005`）
- **初始值**：`0xFFFF`
- **输入反转**：否
- **输出反转**：否
- **异或输出**：`0x0000`

### 8.3 代码实现位置
- **计算函数**：`CalcCRC16()`（`App/Protocol/Src/modbus.c` Line 72-88）
- **校验函数**：`Modbus_ValidateFrame()`（`modbus.c` Line 106-128）
- **响应组包**：所有 `Modbus_Send*()` 函数末尾附加 CRC

## 9. 并发与一致性保证

### 9.1 互斥锁机制
- **锁对象**：`s_modbus_reg_mutex`（CMSIS-RTOS 互斥量）
- **锁粒度**：寄存器级（所有 10 个寄存器共享同一把锁）
- **等待策略**：`osWaitForever`（无限等待，避免忙等）

### 9.2 写操作回调执行顺序
```
// 伪代码流程
Modbus_LockRegisters();          // 加锁
  holding_regs[addr].value = new_value;  // 更新值
  on_change = holding_regs[addr].on_change;  // 提取回调
Modbus_UnlockRegisters();        // 解锁

on_change(old_value, new_value);  // 锁外执行回调（避免死锁）
```

### 9.3 读操作快照机制
```
// 0x03 多寄存器读取
Modbus_LockRegisters();
  for (i = 0; i < reg_count; i++) {
    tx_buffer[...] = holding_regs[start_addr + i].value;  // 快照读取
  }
Modbus_UnlockRegisters();
// 锁外组包 CRC 并发送
```
